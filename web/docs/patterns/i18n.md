# Internationalization (i18n)

Tuai mendukung dua bahasa: **Bahasa Indonesia (default)** dan **English**. Stack: `i18next` + `react-i18next` + lazy-loaded JSON per namespace.

Canonical reference: [fund-transfer feature](../../src/module/finance/features/fund-transfer/).

## Architecture

```
src/locales/
├── i18n-provider.tsx        ← initialize i18next + <I18nextProvider>
├── localization-provider.tsx ← MUI X date picker adapter sync
├── locales-config.ts        ← allLangs, supportedLngs, fallbackLng, resource loader
├── use-locales.ts           ← useTranslate() hook (t, i18n, currentLang, onChangeLang)
├── index.ts                 ← barrel
└── langs/
    ├── id/
    │   ├── common.json      ← shared buttons, state, validation
    │   ├── messages.json    ← toast / system messages
    │   └── fund-transfer.json
    └── en/
        └── ...
```

Provider stack di [src/app.tsx](../../src/app.tsx):

```tsx
<I18nProvider>
  <AuthProvider>
    <SettingsProvider>
      <ThemeProvider>
        <LocalizationProvider>  {/* locale-aware MUI date adapter */}
          {children}
        </ThemeProvider>
      </AuthProvider>
    </SettingsProvider>
  </AuthProvider>
</I18nProvider>
```

## Key rules

1. **Jangan hardcode string user-facing.** Semua label, placeholder, error message, toast, dialog title wajib via `t('...')`.
2. **Default language: `id`.** Tetap tulis key dan JSON `id` dulu, baru `en`.
3. **Namespace per feature.** Feature baru = JSON baru di `langs/id/<feature>.json` dan `langs/en/<feature>.json`. Loader lazy-import sesuai pemakaian.
4. **Shared strings ke `common`.** Contoh: `actions.cancel`, `actions.save`, `actions.delete`, `state.loading`, `pagination.rowsPerPage`.
5. **Persistensi via localStorage key `i18nextLng`.** Tidak ada URL segment.

## Usage in components

```tsx
import { useTranslate } from 'src/locales';

export function MyView() {
  const { t } = useTranslate('fund-transfer');   // namespace-scoped
  const { t: tCommon } = useTranslate('common'); // shared actions

  return (
    <Button>{t('buttons.saveDraft')}</Button>
    <Button>{tCommon('actions.cancel')}</Button>
  );
}
```

## Usage in hooks (non-React scope)

Hook top-level code dan module-level helpers **tidak bisa panggil `useTranslate`**. Import `i18n` langsung dan panggil `i18n.t(...)` dengan fully-qualified key `{namespace}:{key}`:

```ts
import i18n from 'i18next';

export function useEntityList(params) {
  // ...
  const load = useCallback(async (next) => {
    try {
      // ...
    } catch (err) {
      setState({
        ...s,
        error: err instanceof Error ? err.message : i18n.t('fund-transfer:errors.loadData'),
      });
    }
  }, []);
}
```

Trade-off: error message yang sudah ter-set tidak ikut re-translate kalau user switch language. Diterima karena error transient — jarang terjadi dan biasanya di-dismiss cepat.

## Interpolation

Untuk nilai dinamis, pakai double-brace + `values`:

```json
// id/fund-transfer.json
{ "editTransfer": "Ubah {{no}}", "feedback": { "submitted": "{{no}} disubmit untuk persetujuan." } }
```

```tsx
t('editTransfer', { no: 'TRX-001' });        // → "Ubah TRX-001"
t('feedback.submitted', { no: 'TRX-001' });  // → "TRX-001 disubmit untuk persetujuan."
```

Untuk JSX di tengah kalimat (contoh: `<strong>`), pakai `<Trans>`:

```tsx
import { Trans } from 'react-i18next';

<Trans
  i18nKey="fund-transfer:attachment.deleteConfirm"
  values={{ name: file.name }}
  components={{ strong: <strong /> }}
/>
```

## Zod validation messages

Schema harus pakai `t()` lewat factory supaya pesan ikut bahasa aktif:

```tsx
function makeSchema(t: TFunction) {
  return z.object({
    transfer_date: z.string().min(1, { message: t('validation.dateRequired') }),
    amount: z.number().positive(t('validation.amountPositive')),
  });
}

// di component
const { t } = useTranslate('fund-transfer');
const schema = useMemo(() => makeSchema(t), [t]);
```

Pola ini persis dipakai di [fund-transfer-form-dialog.tsx](../../src/module/finance/features/fund-transfer/components/fund-transfer-form-dialog.tsx).

## Dropdown / array of config with labels

Untuk static array (tab config, menu items), simpan `labelKey` bukan `label`, lalu panggil `t(item.labelKey)` di render:

```tsx
const STATUS_TABS = [
  { value: 'draft', labelKey: 'statuses.draft', color: 'info' },
  { value: 'waiting', labelKey: 'statuses.waiting', color: 'warning' },
];

STATUS_TABS.map((tab) => <Tab label={t(tab.labelKey)} ... />)
```

## Helper functions yang butuh locale

Format currency / date harus ikut `i18next.resolvedLanguage`. Contoh di [utils/format.ts](../../src/module/finance/features/fund-transfer/utils/format.ts):

```ts
import i18n from 'i18next';

function currentLocale() {
  return i18n.resolvedLanguage === 'en' ? 'en-US' : 'id-ID';
}

export function formatDate(iso: string) {
  return new Date(iso).toLocaleDateString(currentLocale(), { ... });
}
```

Component yang memakai helper ini harus juga pakai `useTranslate()` agar re-render saat bahasa berganti.

## Language switcher

[LanguagePopover](../../src/layouts/components/language-popover.tsx) sudah otomatis terhubung via `useTranslate()`:
- Sumber list: `allLangs` export dari `src/locales`
- Klik → `onChangeLang(lang)` → set i18next + sync dayjs locale + persist ke localStorage

## Adding a new feature namespace

1. Buat `src/locales/langs/id/<feature>.json` dan `.../en/<feature>.json` dengan struktur paralel.
2. Gunakan `const { t } = useTranslate('<feature>');` di komponen-komponen feature itu.
3. Pindahkan string shared (kalau ada) ke `common.json`, jangan duplikasi.
4. Jangan tambah key ke `fund-transfer.json` untuk feature lain — bikin namespace sendiri.

### Structure template untuk JSON per-feature

Template key groups yang biasanya ada (ikut pola fund-transfer):

```
title              — judul feature / breadcrumb
newEntity          — "Entity Baru"
editEntity         — "Ubah {{no}}"
buttons.*          — label tombol utama
statuses.*         — mapping status enum → label
table.*            — column headers
toolbar.*          — search placeholder, filter labels
rowActions.*       — menu popover item label (View, Edit, Delete, dll)
form.*             — field labels, placeholders, helper text
validation.*       — Zod error messages
confirm.{submit,approve,reject,delete}.{title,confirm,commentLabel,commentPlaceholder}
tabs.*             — detail dialog tab labels
detail.*           — detail dialog field labels + empty states
list.*             — list view empty state
feedback.*         — success/error toast setelah aksi
summary.*          — summary card labels (kalau ada)
errors.*           — hook-level error fallbacks (loadData, loadAttachments, dll)
attachment.*       — attachment-related labels (kalau feature punya file upload)
```

Verifikasi parallel id ↔ en dengan quick script:

```bash
node -e "
const id = require('./src/locales/langs/id/<feature>.json');
const en = require('./src/locales/langs/en/<feature>.json');
const flatten = (o, p='') => Object.keys(o).reduce((a, k) => {
  const path = p ? p + '.' + k : k;
  if (o[k] && typeof o[k] === 'object') Object.assign(a, flatten(o[k], path));
  else a[path] = o[k];
  return a;
}, {});
const idK = Object.keys(flatten(id)).sort(), enK = Object.keys(flatten(en)).sort();
console.log('Missing in EN:', idK.filter(k => !enK.includes(k)));
console.log('Missing in ID:', enK.filter(k => !idK.includes(k)));
"
```
